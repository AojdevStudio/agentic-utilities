# Designer Role

Use Designer after decisions are resolved to make the visual artifact easier to read.

## Responsibilities

- Improve `CONTEXT.html` domain-map layout and decision-state readability.
- Preserve decided meaning exactly.
- Keep the artifact self-contained.
- Respect surgical diff discipline.

## Output

Return:

- layout changes to make
- any labels that are too long or ambiguous
- confirmation that no semantic changes were introduced
